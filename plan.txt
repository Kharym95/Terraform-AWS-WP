CONTEXTE
DESCRIPTION DU PROJET
Ce projet vise à créer une infrastructure AWS pour héberger une application WordPress avec une base de données RDS MySQL.

HYPOTHÈSES, CONTRAINTES ET EXIGENCES GÉNÉRALES
Les hypothèses et contraintes incluent l'utilisation de Terraform pour la gestion de l'infrastructure, AWS comme fournisseur cloud, et l'implémentation de pratiques de haute disponibilité et de sécurité.

ARCHITECTURE TECHNIQUE CIBLE
PRINCIPES GÉNÉRAUX
Utilisation de zones de disponibilité multiples pour la haute disponibilité.
Sécurité renforcée par des groupes de sécurité bien définis.
Utilisation de Terraform pour une gestion de l'infrastructure en tant que code.
ARCHITECTURE LOGIQUE
COMPOSANTS LOGIQUES
Les composants logiques incluent VPC, sous-réseaux, groupes de sécurité, instances EC2, RDS, NAT Gateway, Internet Gateway.

MATRICE D'ÉCHANGE DE DONNÉES
Les échanges de données se feront principalement via HTTP/HTTPS pour les applications web, et MySQL pour la base de données.

ARCHITECTURE PHYSIQUE
DISTRIBUTION ET ORGANISATION DES COMPOSANTS DE LA SOLUTION
Les composants seront organisés en sous-réseaux publics et privés au sein d'un VPC.

FLUX PHYSIQUES ET PROTOCOLES NON-APPLICATIONS
Les protocoles utilisés incluront TCP pour les connexions HTTP/HTTPS et MySQL.

DISTRIBUTION PHYSIQUE DES ENVIRONNEMENTS LOGIQUES
Les environnements seront distribués dans des sous-réseaux publics et privés pour la sécurité et la gestion.

ARCHITECTURE DU SYSTÈME
Comprend les instances EC2 pour l'application WordPress et RDS pour la base de données.

ARCHITECTURE RÉSEAU
Comprend les sous-réseaux, les tables de routage, les passerelles Internet et NAT.

ÉLÉMENTS LOGICIELS (INFRASTRUCTURE)
Les éléments logiciels incluent Terraform pour la gestion de l'infrastructure et les logiciels installés sur les instances EC2.

ÉLÉMENTS MATÉRIELS
Les éléments matériels incluent les types d'instances EC2 et les configurations de stockage.

ARCHITECTURE RÉSEAU
Comprend les composants réseau et les connexions entre eux.

CONTINUITÉ DU SERVICE TECHNIQUE
REDONDANCE
Utilisation de plusieurs zones de disponibilité et sous-réseaux privés et publics pour la redondance.

TOLÉRANCE AUX PANNES
Mise en place de mécanismes pour tolérer les pannes, tels que les sauvegardes automatiques et les configurations de haute disponibilité.

GESTION DES CLÉS DE CHIFFREMENT
Gestion des clés de chiffrement via AWS KMS.

FONCTIONS DE SAUVEGARDE ET DE RÉCUPÉRATION
